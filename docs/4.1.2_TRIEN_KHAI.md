## 1. Mục tiêu triển khai

Sau giai đoạn phát triển trên môi trường cục bộ (`localhost:5000`), hệ thống được triển khai ra môi trường có thể truy cập từ **Internet hoặc mạng nội bộ**, nhằm:

- Minh chứng khả năng vận hành thực tế (PI 4.1 — Triển khai).
- Cung cấp URL demo cho hội đồng / người dùng thử nghiệm.
- Đảm bảo dữ liệu thử nghiệm đạt ngưỡng đề cương (PI 4.2): ≥300 sản phẩm, ≥200 người dùng, ≥500 đơn hàng, ≥300 đánh giá, ≥200 hội thoại AI.

**Công nghệ production:** Python 3.11, Flask 3, Gunicorn, SQLite, scikit-learn (AI), Cloudinary (ảnh upload trên cloud — tùy chọn).

## 3. Phương án 1 — Triển khai lên Render (chi tiết)

### 3.1. Kiến trúc triển khai

```
[GitHub: DeployyWeb]  →  [Render Build]
                              ├── pip install -r requirements.txt
                              ├── python seed.py --reset --fixed  (tạo nongsan.db)
                              └── Upload artifact
                         [Render Runtime]
                              └── gunicorn app:app --bind 0.0.0.0:$PORT --workers 1
```

- **Build:** Cài dependency và **sinh dữ liệu đầy đủ** trong giai đoạn build (tránh seed khi khởi động Gunicorn — gây timeout “No open ports”).
- **Runtime:** Gunicorn phục vụ ứng dụng Flask (`app:app`), cổng do Render inject qua biến môi trường `$PORT`.
- **Cấu hình:** `Procfile`, `runtime.txt` (Python 3.11.9), `render.yaml`, biến môi trường trên Dashboard.

### 3.2. Chuẩn bị mã nguồn

1. Đưa project lên GitHub (repository: `LinhTranDang10072004/DeployyWeb`, nhánh `main`).
2. Không commit file nhạy cảm: `.env`, `.venv/`, `*.db`, `logs/`.
3. Các file phục vụ deploy: `requirements.txt`, `Procfile`, `runtime.txt`, `config.py`, `seed.py`, `app.py`.

### 3.3. Đăng ký và tạo Web Service

1. Truy cập https://render.com, đăng ký bằng tài khoản GitHub.
2. **New → Web Service** → chọn repository `DeployyWeb`.
3. Thiết lập:
   - **Name:** `nong-san-xanh`
   - **Region:** Singapore (Southeast Asia)
   - **Branch:** `main`
   - **Runtime:** Python 3
   - **Instance:** Free

### 3.4. Cấu hình Build & Start

| Hạng mục | Giá trị |
|----------|---------|
| **Build Command** | `pip install -r requirements.txt && python seed.py --reset --fixed` |
| **Start Command** | `gunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 300` |
| **Pre-deploy (Procfile release)** | `python seed.py --reset --fixed` (nếu Render chạy release command) |

**Biến môi trường (Environment):**

| Biến | Giá trị production | Ghi chú |
|------|---------------------|---------|
| `SECRET_KEY` | Chuỗi random 64 ký tự hex | Bảo mật session Flask |
| `FLASK_DEBUG` | `0` | Tắt chế độ debug |
| `PYTHON_VERSION` | `3.11.9` | Khớp `runtime.txt` |
| `LOG_DIR` | `logs` | Ghi log ứng dụng |
| `LOG_LEVEL` | `INFO` | Mức log |
| `CLOUDINARY_*` | (tùy chọn) | Lưu ảnh upload lên CDN khi redeploy |

### 3.5. Triển khai và xác minh

1. **Manual Deploy → Deploy latest commit** (không rollback về `first commit`).
2. Chờ build 5–15 phút; log build hiển thị `[OK] +320 products`, `[DE CUONG] ... -> DAT`.
3. Kiểm tra endpoint sức khỏe:

   `GET https://nong-san-xanh.onrender.com/health`

   Kết quả mong đợi: `"status": "ok"`, `products` ≥ 300, `users` ≥ 200, v.v.

4. Truy cập trang chủ, đăng nhập tài khoản seed:

   | Vai trò | Username | Password |
   |---------|----------|----------|
   | Admin | admin | admin123 |
   | Seller | seller | seller123 |

### 3.6. Lưu ý vận hành trên Render Free

- Instance có thể **tạm dừng** sau khoảng 15 phút không có truy cập; lần mở sau chờ 30–60 giây để khởi động lại.

- Ảnh tĩnh (CSS/JS CDN) và thư mục `static/` được Flask phục vụ; ảnh upload nên dùng **Cloudinary** để không mất sau redeploy.

## 4. Phương án 2 — Deploy VPS (Ubuntu + Nginx + Gunicorn)

Phương án dùng khi cần kiểm soát server hoàn toàn hoặc triển khai lâu dài.

### 4.1. Sơ đồ

```
Internet → Nginx (:80/443) → Gunicorn (127.0.0.1:8000) → Flask app
```

### 4.2. Các bước tóm tắt

1. **Thuê VPS** (Ubuntu 22.04 LTS), mở firewall cổng 80, 443 (và 22 cho SSH).
2. **Cài runtime:** `sudo apt update && sudo apt install python3.11 python3.11-venv nginx`
3. **Clone project:** `git clone <repo-url> /var/www/nongsan`
4. **Virtualenv & dependencies:**
   ```bash
   cd /var/www/nongsan
   python3.11 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
5. **Biến môi trường:** tạo file `.env` (SECRET_KEY, FLASK_DEBUG=0, CLOUDINARY_*).
6. **Khởi tạo DB + seed:**
   ```bash
   python seed.py --reset --fixed
   ```
7. **Gunicorn (systemd):** service chạy  
   `gunicorn app:app --bind 127.0.0.1:8000 --workers 2 --timeout 300`
8. **Nginx:** reverse proxy `proxy_pass http://127.0.0.1:8000;`, cấu hình `client_max_body_size` cho upload ảnh (≤16MB).
9. **HTTPS:** cài chứng chỉ Let's Encrypt (`certbot --nginx`).
10. **Kiểm thử:** truy cập `http://<IP-VPS>` hoặc tên miền từ thiết bị ngoài.

## 5. Phương án 3 — Demo nội bộ qua mạng LAN

Dùng khi bảo vệ đồ án trong phòng lab, không phụ thuộc Internet.

1. Máy chủ: bật Wi‑Fi/LAN, chạy:
   ```bash
   pip install -r requirements.txt
   python seed.py --reset --fixed
   gunicorn app:app --bind 0.0.0.0:5000 --workers 1
   ```
   Hoặc dev: `python app.py` (chỉ nên dùng khi `FLASK_DEBUG=0` trên mạng tin cậy).

2. Lấy IP LAN máy chủ (Windows: `ipconfig`, Linux: `ip a`) — ví dụ `192.168.1.50`.

3. Thiết bị khác (laptop hội đồng, điện thoại) cùng mạng truy cập:  
   `http://192.168.1.50:5000`

4. Tắt firewall chặn cổng 5000 trên máy chủ (Windows Defender / ufw) nếu không ping được.

**Hạn chế:** Không có HTTPS; chỉ dùng demo ngắn, không công khai Internet.
